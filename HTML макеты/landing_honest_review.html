Сильно
Cormorant + IBM Plex Sans — профессиональная пара
Сочетание сериф-дисплея с гротеском работает именно так, как надо для «Tech Astrology». Это одно из лучших типографических решений в нише.
Сильно
Ant Design токены строго соблюдены
CSS-переменные правильно разделены, dark mode переключается без мерцания, семантика цветов не нарушена. Это хорошая основа для будущего React-компонента.
Слабо
SVG-карта — слишком схематична, нет «вау-момента»
Орбиты из тонких линий выглядят как техническая заглушка, а не как продукт. Конкуренты (Astro.com, Co–Star, TimePassages) делают карту визуально плотной и детальной. Первое, что должно произвести впечатление — это карта.
→ Добавить знаки зодиака по кругу, цветовые сектора домов, толстые аспектные линии с разными цветами для разных типов.
Критично
Ноль анимаций при скролле — 2019 год
В 2025 любой приличный лендинг использует scroll-triggered анимации. Карточки должны появляться при скролле (fade-up), SVG-карта должна «рисоваться» при первом появлении (stroke-dashoffset), цифры в stats — считаться анимированно. Без этого страница ощущается статичной и дешёвой.
→ Минимум: IntersectionObserver + CSS transition. Пример: карточки features появляются с задержкой 100ms каждая.
Слабо
Hero-картинка справа — слишком маленькая и изолированная
SVG-карта в Hero занимает ~40% правой колонки, но не доминирует визуально. Современный тренд — карта должна быть «живой»: вращаться, иметь floating-элементы с названиями планет, создавать ощущение интерактивности.
→ Добавить плавающие подписи планет как абсолютно позиционированные элементы с CSS-анимацией float.

Критично
Google Fonts — блокирующий запрос к googleapis.com
Два шрифта грузятся с внешнего CDN синхронно. В России googleapis нестабилен. LCP при медленном соединении может быть 4–6 секунд — это смерть для конверсии (каждая секунда задержки = -7% конверсии по данным Google).
→ Скачать woff2-файлы, положить на свой сервер, подключить через @font-face с font-display: swap. Размер двух шрифтов ~80kb суммарно.
Критично
Один огромный CSS-файл — 500+ строк, всё в <style>
Весь CSS в head-теге одним блоком — это нормально для прототипа, но не для production. Браузер парсит все 500 строк перед первым рендером. При переходе на React нужно CSS Modules или разбивка на critical/non-critical.
→ На текущем этапе: выделить above-the-fold CSS (navbar + hero) в <style>, остальное загрузить асинхронно через <link rel="preload">.
Важно
Нет og:image — Telegram покажет пустой превью
og:image отсутствует. Когда пользователь пошарит ссылку в Telegram-чат, карточка будет без картинки. Для астрологического сервиса, где шеринг — ключевой канал роста, это серьёзный пропуск.
→ Создать статичную og:image 1200×630px с логотипом, названием и SVG-картой. Добавить <meta property="og:image" content="...">.
Важно
Нет Schema.org — потеря расширенных сниппетов
FAQPage и Product с AggregateRating упоминались в концепции, но в коде отсутствуют. Конкуренты с этой разметкой занимают больше места в выдаче Google и получают звёзды рейтинга прямо в сниппете.
→ 20 строк JSON-LD в <head>. FAQPage даёт раскрывающиеся вопросы прямо в поиске.
Хорошо
Минимум JavaScript — без тяжёлых библиотек
Весь интерактив написан на vanilla JS. Без jQuery, без Bootstrap, без лишних зависимостей. Это правильное решение для прототипа — размер скрипта меньше 5kb.

Сильно
Трёхшаговый wizard — правильное решение
Разбивка на шаги с сохранением в localStorage соответствует лучшим практикам 2024–2025. Прогресс-индикатор и персонализация на шаге 2 («Алексей, выберите тариф») работают на конверсию.
Критично
Hero не отвечает на «зачем мне это» — только «что это»
Текущий Hero описывает продукт: «Детальный разбор характера, талантов и жизненных циклов». Но покупатель думает не про функции — он думает про себя: «Почему у меня не идёт карьера?», «Почему я снова в тех же отношениях?». Разрыв между тем что предлагается и тем что ищут — основная причина отказов.
→ Подзаголовок Hero переписать в jobs-to-be-done формате: «Поймите, почему повторяются одни сценарии. Узнайте, когда подходящее время для важных решений. Получите ответы за 60 секунд.»
Критично
Отзывы — нет фото и нет конкретики, выглядят как заглушки
«Описание личности совпало на 90%» — это шаблонный отзыв, который пользователи идентифицируют как сгенерированный ИИ. В 2025 году отзывы без фото, должности и конкретной детали воспринимаются как фейк. Это подрывает доверие.
→ Добавить реальные фото (или нарисованные аватары с характером), конкретную деталь: «Меркурий ретроградный в IV доме — это точно про мои сложности с переездами».
Важно
Блок «Для кого это» отсутствует
Нет сегментации аудитории. Пользователь должен узнать себя в одном из 3–4 персонажей ещё до скролла до тарифов. «Для тех, кто застрял на карьерном плато», «Для пар, которые хотят понять друг друга», «Для тех, кто переживает перемены».
Важно
12 000 карт — цифра без подтверждения
У сервиса ещё нет 12 000 карт. Это число заложено как «желаемое», но при запуске оно будет ложью. Пользователи проверяют дату регистрации домена через whois, смотрят на активность в соцсетях. Несоответствие разрушает доверие моментально.
→ До 1000 карт честнее написать «Первые карты уже рассчитаны · Оцените сами». После — показывать реальное число.
Хорошо
Многошаговая форма правильно откладывает авторизацию
Авторизация только на шаге 3 — это соответствует флоу из документа и снижает барьер входа. Правильное UX-решение.

Где реальное конкурентное окно
Русскоязычный рынок — недообслуженный. Astro.com перегружен, Co–Star без русского языка, местные конкуренты (Астрологи в Telegram) работают без продуктового подхода. Telegram-интеграция как нативный канал — это подлинное преимущество, которого нет ни у кого из топ-игроков. Основная угроза: ИИ-боты в Telegram (GPT-обёртки над астрологическими данными), которые уже появляются. Нужно занять позицию «точность + дизайн» до их масштабирования.
Честно
Это хороший прототип, но ещё не продукт
Текущая реализация — сильная база для согласования структуры и логики. Типографика, цветовая система, wizard-форма и dark mode на уровне 2024–2025. Для разработчика или инвестора — убедительно. Для холодного трафика из рекламы — пока нет.
Главный gap
Нет ни одной анимации — это самый болезненный пробел
В 2025 году отсутствие scroll-анимаций на лендинге — это то же самое, что в 2015 году сайт без адаптивности. Пользователь подсознательно считывает «устаревший» или «дешёвый» продукт ещё до того, как прочитал хоть слово.
Приоритет 1
Анимации: 3 вещи дадут 80% визуального эффекта
SVG-карта рисуется при загрузке (stroke-dashoffset 2 секунды). Карточки features появляются при скролле с задержкой. Цифры в stats считаются от 0. Это ~50 строк JavaScript, но визуально преображает страницу.
Приоритет 2
Шрифты локально + og:image — перед запуском обязательно
Без этих двух вещей нельзя запускать рекламу. Шрифты с googleapis дадут плохой Lighthouse score и медленный старт в России. Отсутствие og:image убьёт шеринг в Telegram — ключевой канал роста.
Приоритет 3
Hero переписать в jobs-to-be-done до первого трафика
Текущий Hero описывает продукт. Нужно описывать боль и результат. Это единственное изменение, которое можно сделать за 20 минут и получить значимый прирост конверсии.
Потом
Реальные отзывы, реальные цифры, Schema.org
До получения первых 50–100 пользователей всё равно не будет реальных отзывов. После запуска собрать скриншоты из Telegram, попросить первых пользователей написать пару слов — и заменить заглушки. Тогда конверсия вырастет заметно.
