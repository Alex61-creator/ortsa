# Политика хранения данных (OPS Data Retention)

Дата: `2026-04-17`

---

## admin_action_logs

| Параметр | Значение |
|---|---|
| Таблица | `admin_action_logs` |
| Срок хранения | **365 дней** (конфигурируется через `ADMIN_LOG_RETENTION_DAYS`, default: 365) |
| Архивация | Записи старше срока хранения можно удалить скриптом или Celery-задачей; перед удалением — при необходимости выгрузить в S3/GCS через `GET /api/v1/admin/orders/{id}/timeline.csv` |
| Доступ к экспорту | Только пользователи с флагом `is_admin = true`; аутентификация через JWT-токен (тот же guard `get_current_admin_user`) |
| Кто может экспортировать | Администраторы системы; DevOps-команда через прямой доступ к БД |

### Примечание по очистке

Плановая очистка не реализована автоматически (нет Celery-задачи). При накоплении >1 млн строк рекомендуется:
1. Добавить Celery beat задачу `purge_old_admin_logs` с интервалом раз в сутки.
2. Либо добавить партицирование по `created_at` (Alembic-миграция, E2 из плана аналитики).
