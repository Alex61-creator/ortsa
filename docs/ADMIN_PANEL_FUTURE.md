# Админ-панель: отложенные функции

Экраны и API ниже **запланированы**, но не реализованы в текущем MVP. В UI админ-SPA они отмечены как заглушки.

## Промокоды

- Нужны сущности в БД: промокод, скидка, лимит использований, привязка к тарифу/периоду.
- API: CRUD, проверка при создании заказа, аудит применений.

## Feature flags

- Хранилище флагов (БД или конфиг с hot-reload).
- Связка с фронтом кабинета и лендинга: условный рендер и A/B без деплоя.

## Воронка и UTM

- Сбор событий на фронте (шаги визарда заказа, авторизация, оплата) с `utm_*` и session id.
- Таблица событий или поток во внешнюю аналитику; агрегаты для экрана «Воронка» в админке.

## Расширенный ROI / LLM

- Учёт токенов и стоимости по заказу (уже частично через LLM-слой) и сведение в отчёты по месяцам.
- Сопоставление выручки тарифа с затратами на генерацию.

## Мониторинг Celery

- Опционально Flower за VPN или read-only API по задачам (отдельно от перезапуска отчёта).

При реализации — обновить OpenAPI-тег «Админка» и убрать заглушки в `frontend-admin`.
